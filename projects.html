ANPR: Vehicle & Driver Verification System

2023-2024 Bondstein Technologies Bangladesh

Overview

End-to-end Automatic Number Plate Recognition system specifically designed for Bengali license plates, integrating vehicle detection, plate reading, and driver face recognition with comprehensive dashboard analytics.

Technical Challenges

  • Low OCR Accuracy on Bangla Plates: Unique fonts and formats with no pretrained models available
  • YOLO Model Failure: Custom YOLO-based models failed on increased variety of Bangla plate numbers
  • Hardware Limitations: PaddleOCR v5 lacked GPU support for specific NVIDIA cards
  • Data Annotation Scale: Required 15,000+ annotations with platform limitations (10K per account)
  • Night-time Visibility: Number plates hardly visible in low-light conditions
  • Camera Integration: RTSP URL configuration and camera angle optimization
  • Speed Detection: Recognition at low frame rates for fast-moving vehicles

Solutions Implemented

  • Custom PaddleOCR Model: Trained specialized model achieving ~94% accuracy on Bengali plates
  • Multi-level Filtering: Syntactic and regional rule-based output cleaning system
  • CPU Optimization: Deployed CPU-optimized version to bypass GPU limitations
  • Annotation Strategy: Created multiple temporary accounts, maintained tracking system for data merging
  • Lighting Setup: Strategic light installation at entry and exit points
  • On-site Data Collection: Manual collection after server setup for model fine-tuning
  • Camera Optimization: Collaborated with IT team for RTSP configuration and angle adjustments

Key Features

Real-time vehicle detection from live RTSP feeds

Custom Bengali OCR with 94% accuracy

Driver face recognition integration

Dashboard with search, filter, and analytics

Cloud and on-premise deployment options

Scalable for ANY country license plates

Technologies Used

PaddleOCR YOLO OpenCV Python RTSP Face Recognition Data Annotation Dashboard Development

Track My Container (KDS Logistics)

2023-2024 KDS Logistics 6000+ Containers

Overview

Real-time container tracking and management system for logistics depot handling 6000+ containers with 3000+ daily operations. Hybrid IoT approach combining GPS, SONAR, height sensors with AI-powered OCR for container ID recognition.

Technical Challenges & Solutions

RTSP Stream Instability

Challenge: RTSP feeds from stacker cranes frequently froze or jittered, especially on Jetson Orin Nano hardware.

Solution: Implemented frame-queueing system with Kalman filtering for stabilization. Tuned Jetson Orin Nano specifically for stream handling optimization.

Fisheye Lens Distortion

Challenge: Severe fisheye distortion from crane-mounted cameras prevented accurate container ID detection.

Solution: Applied pre-calibrated undistortion using OpenCV intrinsic matrix and distortion coefficients.

Vertically Oriented Container IDs

Challenge: Containers stacked with vertical IDs caused OCR to fail completely.

Solution: Built vertical-to-horizontal character rearrangement module (ver_to_hor.py) with YOLO to reassemble text horizontally.

Faded or Damaged Text

Challenge: Old containers had degraded markings leading to OCR misreads.

Solution: Added ISO 6346 validation with intelligent error correction (character substitution, prefix validation).

Gate-Side Infrastructure

Challenge: Camera misplacement, poor LAN connections, inadequate lighting, no backup power.

Solution: Complete infrastructure overhaul: repositioned cameras with site surveys, upgraded LAN cabling, improved lighting, connected to emergency power grid.

System Features

Real-time GPS tracking and movement updates

Hybrid IoT with GPS, SONAR, height sensors

AI camera with automatic OCR reading

Depot space optimization and crane efficiency

Infrastructure-less scalable deployment

ISO 6346 validation and error correction

Technologies Used

Jetson Orin Nano OpenCV Kalman Filtering YOLO OCR GPS/SONAR ISO 6346 RTSP

BAT AI Leaf SOP Checker

2023-2024 Manufacturing British American Tobacco

Overview

AI-powered Standard Operating Procedure compliance system automating leaf buying process monitoring. Ensures strict adherence to SOPs with real-time tracking across bale entry, inspection, and weighing stages.

Complex Technical Challenges

Unique Bale Tracking Across Multiple Steps

Challenge: Bales difficult to track across entry, inspection, and weighing stages with consistent IDs.

Solution: Multi-camera tracker with DeepSORT-inspired IoU matching, ensuring persistent IDs across stages (bale_count.json, state management).

Worker Mis-alerts in Weighing Area

Challenge: Workers frequently entered restricted weighing zones triggering false alerts (80%+ false positive rate).

Solution: Custom detection logic checking IoU overlap between humans and bale zones with persistence rules, reducing mis-alerts by 80%.

RTSP Stream Jitter

Challenge: OpenCV pipeline produced unstable frames affecting detection accuracy.

Solution: Frame queueing system with Kalman filtering for smooth stream processing.

Manual Annotation Inefficiency

Challenge: Extracting bale images for training annotations extremely time-consuming.

Solution: Developed PC-based auto-extraction tool saving annotated frames with timestamps.

Impact & Results

80% False Alert Reduction
100% SOP Compliance Tracking
3 Process Stages Monitored
Real-time Alert System

Unilever Argus Automata (KGF Smart Surveillance)

2023-2024 Unilever 90+ Cameras

Overview

Comprehensive multi-module AI surveillance system featuring intrusion detection, PPE monitoring, vehicle speed tracking, and collision prevention for warehouse safety and security.

Module 1: Intrusion Detection

Challenge: People outside ROIs misdetected as intruders causing alert fatigue.

Solution: Refined logic triggering alerts only when complete bounding box lies inside ROI, eliminating false alarms.

Module 2: PPE Detection System

System Architecture Design

Challenge: Detecting PPE consistently in dynamic warehouses with occlusions and lighting variations.

Solution: Modular YOLOv8 + DeepSORT pipeline with IoU-based PPE-human association and 30-frame temporal persistence achieving >80% MOTA.

Key Optimizations

  • Refactored Tracking: Moved from centroid tracker to DeepSORT, eliminating negative classes
  • Alert Optimization: Prioritized only violations (no-helmet/no-vest) with 5-min cooldown per worker
  • Flicker Elimination: 10-sec consistency buffer with deque smoothing across 300 frames
  • Adaptive Thresholds: Per-class confidence (0.6 for violations, 0.5 for humans)
  • Human-first Filtering: IoU > 0.1 requirement between PPE and person to eliminate false detections
  • SAHI Optimization: Tuned slice size (1280√ó1280), overlap (0.1), TensorRT export for real-time processing

Module 3: Vehicle Detection & Speed Estimation

Technical Innovations

  • End-to-End Pipeline: YOLO + ByteTrack + homography for robust detection, tracking, and speed calculation
  • Jitter Reduction: ByteTrack integration improved IDF1 by 20% for low-FPS feeds
  • Speed Calculation: Kalman filter smoothing + curvature checks removed ~60% noise
  • Perspective Correction: Multi-plane homography with auto-calibration for multi-level warehouses

Module 4: Warehouse Collision Detection

Features: Real-time proximity tracking between forklifts and humans, driver presence detection, idle vehicle logic, speed monitoring (converted to km/h).

Optimization: Median filtering, frame buffers, state persistence timers, per-class confidence thresholds for reduced false positives under occlusion/low-light.

PPE monitoring with >80% MOTA accuracy

Helmet use tracking in stacking alleys

Collision prevention for forklifts and humans

Over-speeding and unsafe behavior detection

Unilever Smart Staircase Safety

2024 Unilever

Overview

AI-powered staircase safety monitoring system detecting unsafe behaviors including improper railing usage and phone usage while climbing stairs using advanced pose landmark analysis.

Technical Challenges & Solutions

Segmentation Failures

Challenge: Overlap between railing and human body confused standard detection algorithms.

Solution: Used pose landmarks (wrists, fingers, ears) to accurately detect railing contact, eliminating segmentation confusion.

Single-Person Limitation

Challenge: Mediapipe tracked only one person at a time in multi-person scenarios.

Solution: YOLO pre-processing detects multiple persons, feeding each separately to Mediapipe for individual pose analysis.
