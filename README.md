## Publish

1. Back up the current repository.
2. Replace the repository root with the contents of this folder.
3. Commit and push to the `main` branch.
4. In GitHub repository settings, confirm **Pages → Deploy from a branch → main / root**.
5. After deployment, test `index.html`, `work.html`, `research.html`, `about.html`, `contact.html`, the résumé link, and mobile navigation.

No admin panel, login, user dashboard, database, Appwrite configuration, API key, build process, or server-side runtime is required.

## Main structure

- `index.html` — executive overview and key impact
- `work.html` — searchable/filterable archive of 24 initiatives
- `research.html` — publications and thesis narrative
- `about.html` — experience, education, skills, and philosophy
- `contact.html` — direct channels only
- `assets/` — CSS, JavaScript, custom SVG visualizations, portraits, and résumé
- `previews/preview-real-visuals.png` — quick review sheet for the new real-project visual style

## Content safety

Project descriptions and visualizations are intentionally generalized. Do not add credentials, IP addresses, model thresholds, internal network diagrams, private client data, access lists, or confidential financial/commercial details.
